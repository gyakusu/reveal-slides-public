Obsidian 特有のコツ — ライブデモ③

4つの便利機能。まずはデイリーノートから始めるのがおすすめ。

1. Templater プラグイン
ノート作成時にテンプレートを自動適用。
<% tp.date.now("YYYY-MM-DD") %> でフロントマターの created を自動入力。

2. デイリーノート
日付ごとに1ノート自動生成（daily/2026-04-10.md）。
その日の考え・読んだPDF・会議メモを全部ここに放り込む。

3. 埋め込み ![[...]]
他ノートをインラインで埋め込む。部分埋め込み（![[note#見出し]]）も可。情報を重複させずに参照できる。

4. Outgoing Links パネル
いま開いているノートが参照している記事の一覧。バックリンク（逆方向）と合わせて、ノートの文脈を全方向から確認できる。「このノートは何に依存し、何から参照されているか」が一目でわかる。

📸 Templater / デイリーノート画面（後ほど差し替え）

コンフリクト対策 — 1ファイル1トピックの原則

ファイルを細分化するだけで、コンフリクトの9割は防げる。

OneDrive 同期 ≠ リアルタイム共同編集
同じ .md を複数人が同時編集するとコンフリクトコピーが発生する。

❌ 悪い例

infra-handbook.md
  （500行、全領域を網羅した巨大ファイル）

✅ 良い例

infra/
├── dns-setup.md
├── dns-failover.md
├── vpn-config.md
└── monitoring.md

3つのルール

  • 1ファイル1トピック — ファイルを細分化。同じファイルを複数人が同時に触る確率を下げる
  • フロー型は Loop に任せる — 議事録・リアルタイムコラボは Loop のまま。ストック型だけ Obsidian
  • SharePoint バージョン履歴を保険に — 最大500バージョン保持。万が一の復元に使う
📸 ファイル細分化前後の比較（後ほど差し替え）

検索の活用 — 二層の検索戦略

書く人は Obsidian、読むだけの人は SharePoint。両方が同じファイルを見ている。

第1層: Obsidian 内検索

Cmd+Shift+F（Mac）/ Ctrl+Shift+F（Win）

  • 全文検索（本文の文字列）
  • タグ検索: tag:#area/infra
  • メタデータ検索: [status:published]
  • 正規表現: /pattern/

第2層: SharePoint 側検索

Obsidian を持っていない人も使える

  • 経営層・他部署メンバーが「ちょっと調べたい」ときに使う
  • Microsoft 365 統合検索（Teams / Outlook と横串）
  • エクスプローラの検索ボックスからも可
📸 Obsidian 検索結果 + SharePoint 検索結果（同じファイルが出る様子）（後ほど差し替え）

セキュリティ・ガバナンス

SharePoint がバックエンドなので、ガバナンスは SharePoint に委譲できる。

権限管理は SharePoint 側で

  • Obsidian には権限の概念がない。Vault を開ける人はすべて見える
  • 部門ごとにライブラリを分ける
  • 機密情報は別ライブラリに隔離し、必要な人だけ同期
  • 1つの巨大Vaultにすべて突っ込まないのがポイント

Wiki に書いてはいけないもの:
個人情報・顧客データ・パスワード
→ パスワード管理ツールや Key Vault に置き、Wiki からはリンクだけ書く

退職者対応:
アカウントを消しても .md ファイルは SharePoint に残る。author フィールドで文脈が辿れる。ファイルはユーザーではなく Vault に帰属する（Loop では不可能）。

まとめ — 明日から始める 3 ステップ

今日の午後にでも始められる。所要時間は 15 分以内。

① Vault を立ち上げる

SharePoint ライブラリを1つ選び、「同期」ボタンを押す。同期先フォルダを Obsidian で「Open folder as vault」。5分で完了。

② 1ノート書く

今日の業務から何か1つ手順や知見をノートに。フロントマターのサンプルをコピペし、title / tags / status だけ埋める。#status/draft で OK。

③ Bases で体感する

Bases にクエリ例をそのまま貼り付ける。#status/draft のノートが一覧に出る。「これは Loop ではできなかった」を自分の手で味わう。

まずは一人で Vault を持ってみるのが最良のスタート。
3ノート書いたら、チームへの展開を考えはじめるタイミング。
